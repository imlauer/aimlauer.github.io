Definición

El comando dd es en realidad una referencia a la instrucción DD del lenguaje de control de trabajos de IBM y significa descripción de datos. El propósito principal de dd es convertir y copiar archivos. Con dd puede copiar fácilmente una partición o un disco completo.

La lista de características incluye:

  • Hacer una copia de seguridad y restaurar una partición o un disco completo
  • Copie regiones de archivos de dispositivos sin formato
  • Convertir formatos de datos
  • Creación de archivos con tamaños fijos

Sin embargo, lo más importante es que el comando dd es la herramienta de creación de imágenes de disco más antigua que aún se encuentra en uso. ¿Por qué todavía está alrededor? Porque es potente, confiable y (lo más importante) fácil de usar.

Lo guiaré a través del proceso de usar dd para crear un clon de una unidad para que luego pueda restaurar esa unidad desde la imagen escrita.

Proceda con precaución

Lo primero que debe saber absolutamente es que invertir los argumentos dentro de un comando dd puede llevar a borrar todos sus datos valiosos. Con ese fin, debe tener cuidado al trabajar con esta herramienta. Asegúrese de conocer la ubicación y los nombres tanto de su fuente como de su objetivo.

También querrá ejecutar el comando dd desde una distribución de Linux en vivo. El comando dd copia cada byte (incluso el espacio no utilizado), por lo que es un proceso intensivo. Trabajar con dd mientras se inicia en una máquina de producción podría dañar los datos (por eso lo ejecutamos desde un disco en vivo). Esto significa descargar su distribución ISO favorita y grabarla en un DVD o en una unidad USB.

Dato importantísimo

La unidad de destino debe ser del mismo tamaño que la unidad que se está clonando o más grande. El comando dd copiará tanto el espacio usado como el no usado del objetivo. Digamos, por ejemplo, que tiene Linux instalado en un disco de 250 GB y actualmente solo usa 100 GB de espacio. El comando dd no distinguirá entre el espacio utilizado y el no utilizado, por lo que la imagen será del disco completo: 250 GB. Puede ayudarse con esto creando una imagen comprimida (más sobre eso en un momento).

Primeros pasos

Lo primero que debe hacer es iniciar su distribución en vivo. No importará qué distribución esté usando, porque casi todas las versiones de Linux incluyen el comando dd. Inicie esa distribución en vivo y luego conecte su unidad externa. La unidad externa que conecte es mejor que sea una unidad vacía y formateada para este propósito.

Copia de seguridad de todo el disco duro a otro DISCO

  1. En el siguiente ejemplo, queremos clonar el disco “sda” y tener un disco idéntico en el servidor como “sdb”. Se puede observar esto usando el comando fdisk -l

  2. Para hacer una copia de seguridad de una copia completa de un disco duro en otro disco duro conectado al mismo sistema, ejecute el comando dd. El nombre del dispositivo UNIX del disco duro de origen es /dev/sda, y el nombre del dispositivo del disco duro de destino es /dev/sdb, la opción de sincronización permite copiar todo mediante la E/S sincronizada.

Aca,
if: unidad de disco de origen (/dev/sda)
of: unidad de disco de destino (/dev/sdb)
bs: leer y escribir BYTES a la vez (el valor predeterminado es 512 bytes, puede usar bs = 64k para discos más grandes)
noerror: continuar después de errores de lectura.
sync: uso Entrada/Salida sincronizado para datos, también para metadatos

sudo dd if=/dev/sda of=/dev/sdb1 bs=64K conv=noerror,sync
  1. Si desea restaurar los datos en el disco original, debe repetir el paso anterior con el origen (sdb) y el destino (sda) correctos. Incluso puede eliminar sda y arrancar desde sdb en este caso.

PRECAUCION

Invertir los argumentos dentro de un comando dd puede llevar a borrar todos sus valiosos datos. Asegúrese de conocer la ubicación y los nombres de su fuente (si =) y su destino (de =).

Copia de seguridad de todo el disco duro a la imagen del disco

  1. Puede crear un archivo de imagen del disco duro y guardarlo en otros dispositivos de almacenamiento. La copia de seguridad de sus datos en una imagen de disco tiene muchas ventajas, una de las cuales es la facilidad de uso. Este método suele ser más rápido que otros tipos de copias de seguridad, lo que le permite restaurar rápidamente los datos después de una catástrofe inesperada. Utilice el siguiente comando para hacer una copia de seguridad del disco duro /dev/sda en un archivo de imagen de disco.
sudo dd if=/dev/sda of=/var/tmp/sda_disk.img
  1. Para restaurar un disco duro con el archivo de imagen de otro disco duro, use el siguiente ejemplo de comando dd.
sudo dd if=sda_disk.img of=/dev/sdc

Copia de seguridad de todo el disco duro a la imagen del disco en NFS y restauración

  1. Compruebe si hay suficiente espacio disponible en el servidor nfs.
sudo showmount -e nfs_server_IP
Export list for xx.xx.xx.xx:
/nfsshare *

sudo df -h
Filesystem                Size   Used  Avail  Use%  Mounted on
xx.xx.xx.xx:/nfsshare     16G    44M   15G    1%    /nfs_test
  1. Crea la imagen. Asegúrese de tener suficiente ancho de banda disponible.
sudo dd if=/dev/sda of=/nfs_test/sda_disk.img
25165824+0 registros en
25165824+0 registros fuera
12884901888 bytes (13 GB) copiados, 263,396 s, 48,9 MB/s

  1. Para restaurar desde la imagen de disco de copia de seguridad en nfs (por ejemplo, si hay una falla de disco en el disco sda y el sistema no puede iniciarse), inicie en modo de rescate con funciones de red.

  2. Monte el recurso compartido nfs. Digamos directorio recién creado /nfsshare.

  3. Restaure sda usando una copia de seguridad de imagen de disco.

sudo dd if=/nfsshare/sda_disk.img of=/dev/sda
